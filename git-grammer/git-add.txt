git add -添加文件/文件夹到暂存区
注意：git不会将空目录或者目录下有被忽略的文件加入到暂存区中

--斜杠前后表示可选择任何一种输入方式
git add -n/--dry-run 文件/文件夹
并没有往暂存区中添加文件/文件夹，而是在提示用户该文件/文件夹是否存在、被忽略等信息

git add -v/--verbose 文件/文件夹
添加文件/文件夹到暂存区，同时提示用户该文件/文件夹的详细信息

git add -f/--force 文件/文件夹
强制添加文件/文件夹到暂存区，即使该文件/文件夹被忽略

git add -i/--interactive 文件/文件夹 
并没有往暂存区中添加文件/文件夹，而是打开指定文件/文件夹的交互式命令环境

git add -p/--patch 文件/文件夹  (较为复杂，未尝试过)
在指定文件中加入指定片段到暂存区

git add -u/--update 文件/文件夹
根据指定文件/文件夹的删除和修改来更新暂存区的内容，但是不会添加新的文件

git add -A/--all/--no-ignore-removal 文件/文件夹
根据指定文件/文件夹的删除、修改、添加来更新暂存区的内容

git add --no-all/--ignore-removal 文件/文件夹
根据指定文件/文件夹的修改、添加来更新暂存区的内容，工作区的删除不会被更新到暂存区

git add -N/--intent-to-add 文件/文件夹
将指定文件/文件夹中的新文件添加到暂存区中，即没有被跟踪的文件。若是已被跟踪的文件则不会进行任何操作

git add --refresh 文件/文件夹
不会向暂存区中添加文件/文件夹，刷新暂存区中文件的stat信息，即一些统计信息

git add --ignore-errors 文件/文件夹
某些文件因索引错误而无法添加，请不要中止操作，而是继续添加其他文件

git add --ignore-missing 文件/文件夹
该选项只能与--dry-run一起使用。检查指定的文件/文件夹是否被忽略

git add --renormalize 文件/文件夹
跟--update一样的效果

git add 文件夹/\*.txt
添加指定目录及子目录下的文件到暂存区
